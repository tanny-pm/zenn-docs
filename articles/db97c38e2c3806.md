# はじめに
2026年4月22日にSwitchBot公式のCLIツール（@switchbot/openapi-cli）が公開されました。

https://x.com/SwitchBotJapan/status/2046792946889146857?s=20

https://github.com/OpenWonderLabs/switchbot-openapi-cli

これまでにPCやWEBサーバーからSwitbotを操作するために利用してきたSwitchBot APIは、認証情報を生成したり、WEB APIを刻子するプログラムを作成するなど、ちょっとしたツールを作るにも少し手間がかかっていました。

今回の公式CLIツールの登場により、ターミナルから直接コマンドを叩くだけで、デバイスの操作やステータス取得が直感的に行えるようになりました。

この記事では、この新しいCLIツールの利用方法と簡単に紹介します。また、お試し用のツールとして**部屋のCO2濃度に応じてカラーライトの色を変化させる可視化ツール**を作ってみたので、その手順も紹介します。

# 環境構築
1. CLIツールのインストール
公式で推奨されているnpmを使った方法で、CLIツールをインストールします。完了すると、ターミナルで`switchbot`コマンドが使えるようになります。

```sh:CLIツールのインストール
$ npm install -g @switchbot/openapi-cli
```

2. トークンとシークレットの設定
CLIからAPIを利用するためには、トークンとシークレットが必要です。
これらはSwitchBotアプリから以下の手順で取得できます。

- SwitchBotアプリを開く > 「プロフィール」 > 「設定」 > 「アプリバージョン」を連続で10回タップ > 出現した「開発者向けオプション」をタップしてトークンとシークレットを取得。
参考：[SwitchBot API公式ドキュメント](https://github.com/OpenWonderLabs/SwitchBotAPI)

取得できたら、ターミナルで設定します。この設定は`~/.switchbot/config.json`に保存されます。

```sh:トークンとシークレットの設定
$ switchbot config set-token <取得したトークン> <取得したシークレット>
```

3. 動作確認
デバイスIDの取得コマンドを実行して、トークンとシークレットが正しく設定できているか確認します。

```sh:デバイスリストの取得
$ switchbot devices list
┌──────────────────────────┬────────────────────┬───────────────────────────────┬──────────┐
│ deviceId                 │ deviceName         │ type                          │ category │
├──────────────────────────┼────────────────────┼───────────────────────────────┼──────────┤
│ 6055Fxxxxxxx             │ ColorLight         │ Color Bulb                    │ physical │
├──────────────────────────┼────────────────────┼───────────────────────────────┼──────────┤
│ B0E9Fxxxxxxx             │ CO2 Sensor         │ MeterPro(CO2)                 │ physical │
├──────────────────────────┼────────────────────┼───────────────────────────────┼──────────┤
```
ここから、対象となる「CO2センサー（MeterPro(CO2))」と「カラーライト（Color Bulb）」の deviceId をメモしておきます。

試しにカラーライトの色を変えてみます。カラーライトの色を変える基本コマンドは以下の通りです。

```sh:カラーライトの色変更
$ switchbot devices command <カラーライトのdeviceId> setColor red
```

CO2センサーの値は以下のように取得できます。`--json`オプションをつけるとJSON形式で取得できるため、他ツールとの連携には便利です。


# CO2濃度可視化ツールの作成と実行

ここまでの設定が完了したら、公式ドキュメントを参考にして、CLIコマンドからSwichbotの情報取得や設定変更をいろいろと実行できます。

    今回は、情報取得と設定変更の例として、1分おきにCO2センサーの数値を読み取り、CO2濃度に応じてカラーライトの色を変更するシェルスクリプトを作成してみました。私は部屋のCO2濃度が高くなってきたら換気をするようにしていますが、センサーの画面表示やアプリの通知に気づかないことが多々あります。部屋のライトの色を変えておけば必ず気づけて便利そうです。

まず、以下のシェルスクリプトを作成して適当な場所に保存しておきます。`SENSOR_ID`と`BULB_ID`は自身のdeviceIDに置き換えてください。

```sh:co2_monitor.sh
#!/bin/bash

# 先ほど取得したデバイスIDを設定
SENSOR_ID="あなたのCO2センサーのID"
BULB_ID="あなたのカラーライトのID"

echo "CO2モニタリングを開始します... (終了するには Ctrl+C)"

while true; do
  # 1. センサーからCO2値を取得
  CO2_VAL=$(switchbot devices status "$SENSOR_ID" --json | jq -r '.data.CO2')

  if [ -z "$CO2_VAL" ] || [ "$CO2_VAL" == "null" ]; then
    echo "[$(date +'%H:%M:%S')] エラー: CO2値が取得できませんでした。"
    sleep 60
    continue
  fi

  # 2. CO2値からグラデーションの色（RGB）を計算
  RGB=$(awk -v co2="$CO2_VAL" '
  BEGIN {
      if (co2 <= 800) { r=0; g=255; b=0 }
      else if (co2 <= 1200) { r=int((co2-800)/400 * 255); g=255; b=0 }
      else if (co2 <= 1600) { r=255; g=int(255 - (co2-1200)/400 * 255); b=0 }
      else { r=255; g=0; b=0 }
      printf "%d:%d:%d\n", r, g, b
  }')

  echo "[$(date +'%H:%M:%S')] 現在のCO2: ${CO2_VAL}ppm -> 設定カラー: ${RGB}"

  # 3. カラーバルブの色を変更
  switchbot devices command "$BULB_ID" setColor "$RGB"

  # 1分（60秒）待機
  sleep 60
done
```

次に、作成したファイルに実行権限を付与して起動します。

```sh
$ chmod +x co2_monitor.sh
$ ./co2_monitor.sh
```

これを実行したままにしておくと、CO2センサーの値に応じてカラーライトの色が変化します。（CO2値の閾値には改善の余地がありそうです）


# まとめ

過去の記事でも紹介してきたように、WEB APIを利用してSwichBotデバイスを操作することは、これまでも可能でした。

https://zenn.dev/tanny/articles/808487545eb30f

しかし、Switchbot CLIを活用すれば、従来のようにゼロからAPIの署名処理やエラーハンドリングを実装する必要がなくなります。

「JSONでデータを取得する」→「シェルのパイプラインで加工する」→「CLIでコマンドを送る」というアプローチで、スマートホームの自動化がサクッと組めるようになっています。複雑なシェルスクリプトも生成AIを使えば簡単に書けるので、ほとんどの自動化用途はSwitchbot CLIで事足りる気がします。

また、今回は利用しませんでしたがMCPサーバーとしての機能も提供されているようです。こちらも色々な用途が考えられるので、試してみたいと思います。
